# راهنمای جامع TunnelBench (تونل‌بنچ)

<p align="right">
  <b>یک ابزار حرفه‌ای CLI برای بنچمارک و تست کیفیت سرورهای VPN در ترافیک‌های سنگین (Heavy Tunnel Usage)</b><br>
  <i>با این ابزار می‌توانید مشکلاتی نظیر اختلال در مسیریابی (Routing)، افت سرعت آپلود و پکت لاس (Packet Loss) بالا را پیش از آنکه کاربران متوجه شوند، شناسایی کنید.</i>
</p>

---

## 📍 این ابزار را کجا نصب کنیم؟ (ایران یا خارج؟)

**پاسخ کوتاه:** حتماً باید روی **سرور ایران** (یا یک کامپیوتر داخل ایران) نصب شود.

**دلیل:**
هدف این ابزار سنجش کیفیت ارتباط بین اینترنت ایران و سرورهای خارجی (از طریق تانل/VPN) است. 
- اگر این ابزار را روی سرور خارج نصب کنید، سرعت و پینگ همان دیتاسنتر خارجی را تست می‌کند که طبیعتاً عالی و بی‌نقص است و هیچ اطلاعات مفیدی درباره وضعیت فیلترینگ یا مسیر ایران به شما نمی‌دهد.
- اما وقتی آن را روی **سرور ایران** نصب می‌کنید، ابزار دقیقاً مانند یک کاربر ایرانی رفتار می‌کند. شما در فایل کانفیگ، پروکسی سرورهای خارجی خود را به آن می‌دهید و ابزار سعی می‌کند از طریق آن پروکسی‌ها دیتا را دانلود و آپلود کند. این کار باعث می‌شود کیفیت واقعی تانل شما و تاثیر اختلالات روی آن دقیقاً اندازه‌گیری شود.

## 📝 نحوه پیکربندی (کانفیگ)

برای معرفی سرورهای خود به ابزار، یک فایل به نام `config.yaml` در پوشه‌ای که برنامه را اجرا می‌کنید بسازید.

```yaml
# config.yaml
servers:
  - name: Germany-V2ray
    host: 104.18.2.1
    proxy_url: socks5://user:pass@127.0.0.1:1080
    
  - name: Finland-Direct
    host: 104.18.4.1
```
* **name**: یک نام دلخواه برای سرور.
* **host**: آی‌پی یا دامنه سرور (برای گرفتن پینگ و پکت لاس استفاده می‌شود).
* **proxy_url**: (اختیاری) اگر می‌خواهید سرعت دانلود و آپلود از داخل تانل یا پروکسی شما عبور کند، پورت لوکال (مثلا کلاینت Xray یا SSH) خود را اینجا وارد کنید.

## 🚀 راهنمای استفاده و دستورات

این ابزار شامل چندین دستور حرفه‌ای برای مدیریت تست‌ها است:

### ۱. اجرای بنچمارک (Benchmark)
این دستور تک‌تک سرورهای داخل فایل کانفیگ را بررسی کرده، سرعت و پینگ آن‌ها را می‌گیرد و در نهایت یک جدول امتیازدهی با گرافیک بسیار زیبا به شما نمایش می‌دهد.
```bash
# تست تمام سرورهای داخل config.yaml
tunnelbench benchmark

# تست فقط یک یا چند سرور خاص
tunnelbench benchmark Germany-V2ray
```

### ۲. مقایسه با تست قبلی (Compare)
فرض کنید سرور شما دیشب خوب بوده اما امروز کاربران از افت سرعت شاکی هستند. با این دستور می‌توانید تست الان را با تست قبلی مقایسه کنید تا دقیقاً ببینید پینگ چقدر بالا رفته یا سرعت چقدر افت کرده است.
```bash
tunnelbench compare Germany-V2ray
```

### ۳. تولید گزارش برای مدیران (Report)
اگر می‌خواهید گزارش وضعیت سرورها را آرشیو کنید یا برای کسی بفرستید، این دستور علاوه بر تست، نتایج را در قالب‌های `HTML`, `JSON`, `Markdown` و `TXT` ذخیره می‌کند.
```bash
tunnelbench report
```
گزارش‌ها در پوشه `reports/` ذخیره می‌شوند. نسخه HTML دارای قالب جذاب Dark Mode است!

### ۴. مشاهده تاریخچه تست‌ها (History)
تمامی تست‌هایی که تا به حال گرفته‌اید در یک دیتابیس سبک (SQLite) ذخیره می‌شوند. برای مشاهده سوابق:
```bash
# دیدن ۱۰ تست آخر کل سرورها
tunnelbench history

# دیدن تست‌های ۵ روز اخیر یک سرور خاص
tunnelbench history Germany-V2ray --limit 5
```

## 🧠 سیستم امتیازدهی چگونه کار می‌کند؟

ابزار از نمره پایه **۱۰۰** شروع می‌کند و بر اساس فاکتورهای زیر نمره کسر می‌کند:
1. **پکت لاس (Packet Loss)**: مرگبارترین مشکل برای VPN است. به ازای هر درصد پکت لاس، نمره زیادی کسر می‌شود. اگر پکت لاس بالای ۵٪ باشد، سرور مستقیماً نمره صفر و ۱ ستاره می‌گیرد.
2. **پینگ (Ping)**: پینگ‌های بالای ۱۰۰ الی ۱۵۰ میلی‌ثانیه باعث افت نمره می‌شوند.
3. **جیتر (Jitter)**: نوسان در پینگ به شدت روی گیمینگ و تماس‌های صوتی تاثیر می‌گذارد و جریمه خاص خود را دارد.
4. **سرعت (Speed)**: سرعت آپلود و دانلود نسبت به یک شبکه ایده‌آل سنجیده شده و نمره‌دهی می‌شود.

در نهایت نمره نهایی تبدیل به ستاره (`★★★★★` تا `☆☆☆☆☆`) می‌شود تا در یک نگاه وضعیت سرور را تشخیص دهید.
